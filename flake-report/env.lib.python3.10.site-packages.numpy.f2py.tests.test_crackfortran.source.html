172 class TestDimSpec(util.F2PyTest):
173 """This test suite tests various expressions that are used as dimension
174 specifications.
175  
176 There exists two usage cases where analyzing dimensions
177 specifications are important.
178  
179 In the first case, the size of output arrays must be defined based
180 on the inputs to a Fortran function. Because Fortran supports
181 arbitrary bases for indexing, for instance, `arr(lower:upper)`,
182 f2py has to evaluate an expression `upper - lower + 1` where
183 `lower` and `upper` are arbitrary expressions of input parameters.
184 The evaluation is performed in C, so f2py has to translate Fortran
185 expressions to valid C expressions (an alternative approach is
186 that a developer specifies the corresponding C expressions in a
187 .pyf file).
188  
189 In the second case, when user provides an input array with a given
190 size but some hidden parameters used in dimensions specifications
191 need to be determined based on the input array size. This is a
192 harder problem because f2py has to solve the inverse problem: find
193 a parameter `p` such that `upper(p) - lower(p) + 1` equals to the
194 size of input array. In the case when this equation cannot be
195 solved (e.g. because the input array size is wrong), raise an
196 error before calling the Fortran function (that otherwise would
197 likely crash Python process when the size of input arrays is
198 wrong). f2py currently supports this case only when the equation
199 is linear with respect to unknown parameter.
200  
201 """
